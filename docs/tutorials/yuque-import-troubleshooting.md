# 语雀 Markdown 导入排查指南

语雀导入建议按三个步骤进行：生成计划、单篇测试、批量导入。

## 生成计划

生成计划不会写入语雀，只会扫描本地 Markdown 目录，并检查目标知识库是否可访问。

计划报告会显示：

- `totalDocs`：待导入 Markdown 数量。
- `folderCount`：本地文件夹数量，导入时会尽量保留为语雀目录。
- `localImageCount`：能找到并准备上传的本地图片。
- `missingLocalResourceCount`：Markdown 引用了但本地找不到的资源。
- `remoteImageCount`：远程图片链接，通常不会重新上传。
- `largeImageCount`：较大的本地图片，可能需要提高上传超时。

## 图片没有显示怎么办

先打开 `00-语雀导入计划.json` 或 `00-语雀导入报告.json`。

- 如果是本地图片缺失，需要把图片文件放回 Markdown 引用路径。
- 如果是远程图片，确认该图片是否能公开访问。
- 如果是上传超时，可以提高“图片上传超时秒”和“失败重试次数”。

## 导入失败怎么办

导入完成后如果有失败项，可以点击“打开报告”查看具体文件，再点击“重试失败”只重试失败文档。
