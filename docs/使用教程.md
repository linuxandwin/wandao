# 使用教程

## 1. 启动万能导

```powershell
python wandao.py
```

选择要导出的知识库类型，然后点击“打开导出界面”。

也可以导入仓库里的 Codex Skill，让 AI 帮你一键启动：

```powershell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.codex\skills" | Out-Null
Copy-Item -Recurse -Force ".\skills\run-wandao" "$env:USERPROFILE\.codex\skills\run-wandao"
```

导入后，新开一个 AI 对话，说：

```text
Use $run-wandao 帮我分析这个知识库链接，推荐导出参数并启动万能导：<你的链接>
```

这个 Skill 会直接调用 `skills/run-wandao/scripts/launch_wandao.py`，再由这个脚本启动 `wandao.py`。如果你没有提前克隆项目，脚本会尝试自动从 GitHub 拉取万能导。

如果你不知道 URL 深度、是否跳过视频页、请求延迟该怎么填，可以直接让 AI 根据链接和导出目标推荐。

## 2. 第一次登录

每个平台第一次使用都需要保存一次登录凭证：

1. 填写入口 URL。
2. 点击“登录并保存凭证”。
3. 浏览器打开后手动登录。
4. 回到工具窗口点击确认。

凭证文件保存的是 Cookie，不保存账号密码。请不要提交到 Git，也不要分享给他人。

新电脑上运行时，工具会自动查找 Chrome、Edge 或 Chromium 的常见安装位置，也会检查 PATH。若浏览器安装在非常规位置，可以设置 `WANDAO_BROWSER` 环境变量为浏览器可执行文件路径。

每个导出界面都有“浏览器程序路径”：

- 点击“查找”可以自动扫描本机浏览器。
- 点击“选择”可以手动指定浏览器程序。
- 如果没有找到浏览器，请先安装 Chrome、Edge 或 Chromium，再重新点击“查找”。

## 3. 选择目录

点击“读取目录”后，工具会展示目录树：

- 双击目录或文档可以切换选择。
- “全选”会选择全部文档。
- “全不选”会清空选择。
- “反选”会反转当前选择。

读取目录后，导出按钮只处理已勾选内容。未读取目录时，默认导出全部可识别内容。

## 4. 增量导出

点击“增量导出选中/全部”。

增量模式会识别本地 Markdown 底部的来源标记，跳过已经导出的文档，只补缺失内容。

## 5. 全量导出

点击“全量导出选中/全部”。

全量模式会重新打开并覆盖对应文档，适合远端内容更新后重新同步。

## 6. 请求延迟

默认参数：

```text
请求延迟秒: 0.8
请求随机浮动秒: 0.4
```

表示每次打开文档或请求正文前等待 `0.8 + 0~0.4` 秒。建议保留默认值或适当调大，不建议调成 0。

知识星球导出默认 URL 深度是 `2`，也就是会导出目录文章本身，并继续进入正文里的下一层知识星球链接。

## 7. 停止任务

导出过程中可以点击“停止当前任务”。工具会在当前文档或图片处理完成后的安全点停止，并保留已经导出的文件和报告。

## 8. 输出目录

默认输出在：

```text
exports/
```

你可以在 GUI 中选择其他目录。建议不要把导出的内容提交到公开仓库。

## 9. 配合 AI 学习代码项目

万能导导出的 Markdown 文档可以和源码项目放在一起，让 AI 同时阅读“教学文档 + 真实代码”。

推荐做法：

1. 用万能导导出你有权限访问的教学文档。
2. 把导出的目录复制到源码项目中，例如：

```text
your-project/
  src/
  docs/
  exported-knowledge/
    00-知识库入口.md
    01-项目介绍.md
    ...
```

3. 用 AI 工具打开整个 `your-project/` 目录。
4. 把 `prompts/项目学习导师提示词.md` 里的提示词发给 AI。
5. 让 AI 按“先读教学文档、再读源码、最后结合业务流程讲解”的方式带你学习。

这种用法适合学习带有源码和配套教学文档的项目，可以减少自己在文档、代码、业务流程之间来回切换的成本。
