# 万能导插件开发指南

万能导 1.2.0 开始支持文件型 provider。一个 provider 可以是自动化脚本，也可以只是教程说明，还可以是“部分自动化 + 部分教程”的混合形态。

## Provider 类型

- `automation`：主程序渲染表单并执行 Python 脚本。
- `guide`：不执行脚本，只在平台页展示 `README.md` 教程。
- `hybrid`：既展示教程，也提供可执行动作。

## 协议与合规

万能导采用 AGPL-3.0-only 协议开源。提交 provider、脚本或文档时，默认表示你同意按本项目协议授权贡献内容。

如果实现参考了第三方项目，请在 PR 中说明来源和许可证。不要直接复制许可证不兼容的代码、图片或文档；平台自动化也不能绕过登录、权限校验或访问控制。

## 目录结构

```text
providers/
  your-provider/
    provider.json
    README.md
    export.py
```

开发模板在：

```text
providers/_template/
```

以下划线开头的目录不会被自动加载。

## provider.json 核心字段

```json
{
  "schemaVersion": 1,
  "id": "your-provider",
  "name": "你的平台",
  "title": "你的平台导出",
  "navLabel": "你的平台",
  "description": "一句话说明这个 provider 做什么",
  "type": "hybrid",
  "group": "export",
  "trustLevel": "community",
  "status": "experimental",
  "guide": "README.md",
  "requirements": {},
  "toc": {},
  "capabilities": {
    "export": true,
    "import": false,
    "guide": true,
    "images": true,
    "tree": false,
    "batch": true
  },
  "fields": [],
  "actions": []
}
```

## 信任等级

`trustLevel` 用来告诉用户这个 provider 的维护来源和风险级别：

- `official`：官方内置维护。
- `community`：社区贡献，随项目一起发布。
- `local`：用户本机自行放入的 provider。
- `experimental`：实验性能力，可能随平台变化失效。

建议社区 PR 默认使用：

```json
{
  "trustLevel": "community",
  "status": "experimental"
}
```

功能稳定后再把 `status` 调整为 `beta` 或 `stable`。

## 依赖声明

插件如果需要额外依赖，可以在 `requirements` 中声明：

```json
{
  "requirements": {
    "python": ["requests>=2.31.0", "beautifulsoup4>=4.12.0"],
    "system": ["Chrome 或 Edge"],
    "notes": ["首次使用需要登录目标平台。"]
  }
}
```

当前版本会在 UI 中展示依赖提示。后续版本会继续补自动检测和安装流程。

## UI 字段

`fields` 用来声明 UI 表单。主程序会自动渲染，不需要社区开发者写 Electron 前端。

支持字段类型：

- `text`
- `password`
- `number`
- `textarea`
- `directory`
- `file`
- `checkbox`
- `select`
- `notice`

示例：

```json
{
  "name": "output",
  "label": "输出目录",
  "type": "directory",
  "arg": "--output",
  "required": true
}
```

用户填写后，主程序会把它转换成命令行参数：

```text
--output 用户选择的目录
```

## 动作按钮

`actions` 用来声明按钮和脚本：

```json
{
  "id": "export",
  "label": "开始导出",
  "script": "export.py",
  "args": ["--progress-every", "1"],
  "actionName": "导出",
  "confirm": "确认开始导出吗？"
}
```

脚本路径只能指向 provider 目录内部的 `.py` 文件，不能使用 `../` 跳出目录。

## 通用目录树协议

如果插件支持“读取目录 -> 勾选部分内容 -> 导出/导入选中项”，请声明：

```json
{
  "capabilities": {
    "scanToc": true
  },
  "toc": {
    "itemsPath": "nodes",
    "idKey": "nodeId",
    "exportIdKey": "exportId",
    "titleKey": "title",
    "parentIdKey": "parentNodeId",
    "selectableKey": "selectable",
    "selectionArg": "--doc-id"
  }
}
```

读取目录动作建议这样写：

```json
{
  "id": "scan",
  "kind": "scan",
  "label": "读取目录",
  "script": "export.py",
  "args": ["--scan-toc"],
  "includeSelection": false
}
```

脚本返回：

```json
{
  "provider": "your-provider",
  "nodes": [
    {
      "nodeId": "folder-1",
      "exportId": "",
      "title": "文件夹",
      "parentNodeId": "",
      "selectable": false
    },
    {
      "nodeId": "doc-1",
      "exportId": "doc-1",
      "title": "第一篇文档",
      "parentNodeId": "folder-1",
      "selectable": true
    }
  ]
}
```

用户勾选后，后续动作会自动追加：

```text
--doc-id doc-1
```

如果你的脚本使用其他参数名，可以设置：

```json
{
  "toc": {
    "selectionArg": "--item-id"
  }
}
```

## 动态动作与字段回填

有些平台需要先读取空间、知识库或文件夹，再填充下拉框。可以用 `updates`：

```json
{
  "id": "list-spaces",
  "label": "读取空间列表",
  "script": "export.py",
  "args": ["--list-spaces"],
  "includeSelection": false,
  "updates": [
    {
      "field": "target_space",
      "type": "options",
      "path": "spaces",
      "valueKey": "id",
      "labelKey": "name",
      "placeholder": "请选择目标空间"
    }
  ]
}
```

脚本返回：

```json
{
  "spaces": [
    {"id": "space_1", "name": "默认空间"}
  ]
}
```

UI 会自动把 `target_space` 这个 `select` 字段更新为返回的选项。

如果只是把返回值写入普通输入框：

```json
{
  "updates": [
    {
      "field": "space_id",
      "path": "data.spaceId"
    }
  ]
}
```

## 脚本输出

脚本可以持续输出普通日志。任务结束时，最后输出 JSON，便于万能导记录任务历史：

```json
{
  "provider": "your-provider",
  "mode": "export",
  "totalDocs": 10,
  "exportedDocs": 10,
  "failureCount": 0,
  "failures": []
}
```

进度日志建议使用：

```text
progress 3/10 exported=3 skipped=0 failures=0
```

## 教程型 Provider

如果平台已经自带导出能力，或者暂时不适合自动化，可以只提交：

```text
provider.json
README.md
```

并设置：

```json
{
  "type": "guide",
  "group": "guide"
}
```

这种 provider 会出现在“平台中心”里，用户进入平台后可以查看教程。

## 共创建议

- 优先说明平台限制、登录方式和权限要求。
- 先做可维护的最小功能，不要强行模拟复杂点击。
- 能走官方 API 或稳定接口时，优先 API 化。
- 无法稳定自动化的平台，可以先提交教程型 provider。
- 不要提交明文密钥、个人 Cookie、测试账号或平台私有数据。
