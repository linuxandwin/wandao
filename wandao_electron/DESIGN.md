# Wandao Desktop Design System

## Product Character

万能导是本地优先的知识迁移工具。界面要让非技术用户清楚知道三件事：从哪里开始、任务正在做什么、失败后怎么继续。

视觉方向借鉴 Wise 的亮绿、友好和清晰，但不复制其品牌标识。万能导自己的识别元素是“选择平台 -> 执行任务 -> 本地 Markdown”的知识迁移路径。

## Core Principles

1. 每个页面只突出一个主操作。登录、检测、读取目录、打开文件等都是次级操作。
2. 亮绿只表示品牌与主操作，不兼任成功状态。成功、警告、错误使用各自语义色。
3. 用表面色差建立层级，避免大面积装饰渐变和重阴影；应用外壳允许使用低对比渐变连接导航与内容。
4. 文案直接说明用户下一步，不展示内部架构或 Provider 实现细节。
5. 标准 UI 和自定义 Provider UI 共用同一套令牌与基础组件。

## Color Tokens

| Role | Light | Purpose |
| --- | --- | --- |
| Brand | `#9fe870` | 主按钮、当前导航、关键标记 |
| Ink | `#0e0f0c` | 标题、深色外壳 |
| Deep green | `#163300` | 品牌色上的深色文字、正向强调 |
| Canvas | `#e8ebe6` | 页面工作区 |
| Surface | `#ffffff` | 卡片、表单、阅读区 |
| Soft surface | `#f3f5f1` | 次级卡片、列表行、折叠区 |
| Success | `#2ead4b` | 完成状态 |
| Warning | `#b86700` | 等待、风险、中断 |
| Danger | `#c62931` | 失败、停止、破坏性操作 |

暗色模式保留同一亮绿主色，页面背景改为深绿黑，正文至少满足 WCAG AA 对比度。

## Typography

- 标题：`Aptos Display` / `Segoe UI Variable Display`，中文回退到 `PingFang SC` / `Microsoft YaHei UI`。
- 正文：`Aptos` / `Segoe UI Variable Text`，中文使用同一系统回退。
- 日志与代码：`Cascadia Mono` / `SFMono-Regular` / `Consolas`。
- 大标题使用 800-900 字重；正文通常 400-500；按钮和标签使用 700 左右。

## Shape And Spacing

- 基础间距为 4px，常用间距为 8 / 12 / 16 / 24 / 32 / 40px。
- 卡片主圆角 24px，输入框 12px，按钮使用胶囊圆角。
- 常规按钮和输入框高度至少 46px；触控目标不得小于 44px。
- 默认卡片无阴影。只有悬浮操作条、粘性切换器和需要脱离页面的浮层使用柔和阴影。

## App Shell

- 日间顶栏和侧栏使用低对比蓝灰到浅绿渐变，夜间使用深灰蓝到深灰绿渐变，壳层与内容之间保留柔和边界。
- 当前导航使用中性抬升面、品牌色细线和图标强调，不使用整块高饱和色。
- 主内容使用轻微渐变的工作区背景和清晰内容面，圆角只用于表达外壳与工作区的层级。
- 首页 Hero 跟随主题使用柔和渐变，并展示知识迁移路径组件，避免日间模式出现突兀的大面积纯黑块。

## Components

### Buttons

- `.btn-primary`：当前页面唯一主操作，亮绿背景、近黑文字。
- `.btn-secondary`：登录、检测、浏览、读取目录等辅助操作，白色或柔和表面。
- `.btn-danger`：停止或破坏性操作，默认描边，悬停后才增强红色背景。
- `.btn-text`：低优先级链接动作，例如刷新、关于和复制。

### Forms

- 标签必须始终可见，不用 placeholder 代替标签。
- 高级参数放入 `.advanced-section`，默认折叠。
- 帮助文本紧跟字段，错误信息使用语义色并给出下一步。
- 多按钮操作区统一使用 `.action-section`，主操作不超过一个。

### Tasks And Logs

- 任务中心显示状态、时间、摘要、失败预览和可恢复动作。
- 日志默认折叠，任务开始时自动展开。
- 用户日志提供可理解的信息，详细日志保留定位错误所需的技术内容。

### Provider Extensions

- Provider 可以使用标准字段和动作自动生成 UI。
- 自定义 UI 只能渲染在 Provider 内容区，不修改全局导航、顶栏或其他平台页面。
- 自定义 UI 应复用现有类名和颜色令牌，不引入新的品牌色。
- 平台卡片只展示“导出 / 导入 / 教程”三类能力；重试、目录、图片等能力在平台内部展示。

## Accessibility And Motion

- 所有操作必须可通过键盘完成，并具有清晰的 `:focus-visible` 状态。
- 正文对比度至少 4.5:1。
- 异步任务必须显示加载、进度、完成或失败反馈。
- 动画持续 150-250ms，仅用于状态和层级变化。
- 必须支持 `prefers-reduced-motion: reduce`。

## Responsive Rules

- `>= 1120px`：完整侧栏，首页双栏。
- `900-1119px`：收窄侧栏，首页单栏，平台两列。
- `< 760px`：导航改为顶部横向列表，内容单列，操作按钮换行。
- 所有 Provider 表单在 900px 最小桌面窗口内不得出现横向滚动。
