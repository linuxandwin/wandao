# Wandao 桌面端使用说明

## 普通用户

普通用户优先下载 Releases 页面里的发行版：

- Windows：下载 `.exe` 安装包或便携版。
- macOS：下载 `.zip` 压缩包，解压后运行应用。

当前桌面端仍会调用本机 Python 后端，请先安装 Python 3.10+。使用印象笔记导出时，还需要在本机 Python 中安装 `evernote-backup==1.13.1`。

## 源码运行

```bash
cd wandao_electron
npm install
npm start
```

## 导出流程

1. 选择左侧平台。
2. 填写知识库入口 URL。
3. 点击“登录并保存凭证”，在浏览器完成登录后回到工具确认。
4. 点击“读取目录”，在目录树里选择要导出的内容。
5. 按需调整“增量导出”、请求延迟、评论区等选项。
6. 点击“开始导出”。

未读取目录时，工具默认导出入口下可识别的全部内容。

印象笔记不需要入口 URL。第一次使用时填写账号和密码，点击“登录并同步”，再读取目录、勾选范围并导出。

## 飞书 Markdown 导入

1. 填写目标飞书 Wiki URL 和本地 Markdown 目录。
2. 打开飞书开放平台，创建企业自建应用。
3. 填写 App ID / App Secret，并点击“保存 API 配置”。
4. 点击“初始化开放平台权限”，按页面提示开通权限并发布应用新版本。
5. 点击“检查应用身份”。
6. 点击“授权目标 Wiki 文档应用”，把当前应用加入目标 Wiki 的文档应用列表并设为可编辑。
7. 点击“登录并保存凭证”。
8. 点击“探测目标 Wiki”和“生成计划”。
9. 先做“单篇导入测试”，确认正常后再“批量导入”。

如果遇到 `99991672`，通常是开放平台 API scope 未开通或未发布新版本。

如果遇到 `131006` 或 `no destination parent node permission`，通常是目标 Wiki 父页面没有给当前企业自建应用文档应用写入权限。

## 进度与停止

任务运行时，界面会显示全局进度条和底部运行日志。导出和批量导入会尽量显示已处理数量、百分比和剩余时间。

需要中断时点击“停止”，工具会在安全点结束并保留已完成文件。

## 打包

```bash
npm run build:win
npm run build:mac
```

Windows 包可在 Windows 本机生成。macOS 包建议在 macOS 或 GitHub Actions 的 macOS runner 生成。
